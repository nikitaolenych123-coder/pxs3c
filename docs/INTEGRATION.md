# Інтеграція зовнішніх рушіїв (RPCS3)

Цей проєкт не містить коду RPCS3. Додаємо лише міст (bridge) для динамічного завантаження, щоб не порушувати ліцензії й уникнути копіювання.

## Варіант A: Динамічний міст
- Зібрати RPCS3 як спільну бібліотеку з експонованими C-API:
  - `int rpcs3_init();`
  - `int rpcs3_load_elf(const char* path);`
  - `void rpcs3_run_frame();`
  - `void rpcs3_shutdown();`
- Покласти `librpcs3_bridge.so` або `librpcs3.so` поряд із APK/бінарем.
- `Rpcs3Bridge` викликає ці символи через `dlopen/dlsym`.

### Android
- Переконайтеся, що `pxs3c_core` лінкується з `libdl` (вже налаштовано).
- Розмістіть `.so` у доступному для додатку шляху (наприклад, поряд із native libs у APK або у внутрішньому сховищі й завантажуйте з абсолютного шляху).
- Виклик `Emulator.loadGame("/sdcard/...")` спробує завантажити ELF/SELF через RPCS3 міст.

## Варіант B: Сабмодуль
- Додати RPCS3 як git submodule у `external/rpcs3` (за згодою з ліцензією GPL):
  - Весь додаток стає GPL-сумісним; вихідний код має бути доступним при розповсюдженні.
- Побудова адаптерів, які посилаються на внутрішні API RPCS3 — потребує узгодження структури.

## Ліцензії
- RPCS3 — GPL (перевірити точну версію). Розповсюдження Android APK, що включає код RPCS3, вимагає надання повного вихідного коду і відповідності GPL.
- Уникайте копіювання будь-яких фрагментів коду без перевірки; використовуйте чіткі межі API.

## Рекомендації
- Почати з варіанта A для експериментів.
- Якщо переходити до B — підготуйте публічний репозиторій із повним вихідним кодом і атрибуцією.
